# 要件定義書 - パーソナルカラー診断アプリ

## 1. 目的

小学5年生を主なターゲットユーザーとして、スマートフォンのカメラで撮影した顔写真から、AI技術を活用してパーソナルカラー（イエベ・ブルベ）を診断し、友達との会話のきっかけとなるエンターテイメントアプリを開発する。

## 2. 機能要件

### 2.1 必須機能

- [ ] **カメラ撮影機能**
  - 前面カメラ（セルフィー）での顔写真撮影
  - iPhoneの標準解像度での撮影対応
  - カメラアクセス許可の適切な処理

- [ ] **パーソナルカラー診断機能**
  - 撮影した顔写真をサーバーに送信
  - AI（Vertex AI Gemini-2.5-pro）による画像解析
  - イエベ・ブルベの判定結果取得

- [ ] **診断結果表示機能**
  - イエベ・ブルベの診断結果表示
  - 診断理由の表示
  - 視覚的で分かりやすいUI表示

- [ ] **基本的なエラーハンドリング**
  - カメラアクセス拒否時のエラー表示
  - ネットワークエラー時のエラー表示
  - サーバーエラー時のエラー表示

### 2.2 オプション機能（将来実装候補）

- [ ] 撮影ガイド機能（顔位置の補助表示）
- [ ] 4シーズン分類・16タイプ分類の詳細診断
- [ ] 画像の明度・色温度補正
- [ ] 診断履歴の保存機能
- [ ] SNSシェア機能
- [ ] おすすめアイテム表示

## 3. 非機能要件

### 3.1 パフォーマンス

- 診断処理時間: 10秒以内
- アプリ起動時間: 3秒以内
- 画像アップロード: ネットワーク状況に依存するが、適切なローディング表示

### 3.2 セキュリティ

- 撮影画像はサーバーに保存しない（一時的な解析処理のみ）
- 個人情報の収集・保存は行わない
- HTTPS通信によるデータ暗号化

### 3.3 保守性

- Clean Architectureの適用
- 単体テスト・統合テストの実装
- 明確なコードドキュメント

### 3.4 互換性

- iOS対応（iPhone）
- App Store 4+レーティング対応
- 日本語のみ対応

## 4. 制約事項

### 4.1 技術的制約

- **クライアント**: Flutter（iOS）
- **サーバー**: ADK + Agent Engine + Vertex AI
- **AI**: Gemini-2.5-pro
- **対象OS**: iOS（iPhoneのみ）
- **ネットワーク**: オンライン必須

### 4.2 ビジネス制約

- **開発期間**: 2025年8月14日〜9月15日（約1ヶ月）
- **リリース予定**: 2025年9月15日
- **予算**: 無料アプリ（収益モデルなし）
- **対象地域**: 日本のみ

### 4.3 UI/UX制約

- **ターゲットユーザー**: 小学5年生レベルの操作性
- **文字情報**: 最小限に抑制
- **操作**: 直感的で簡単な操作フロー

## 5. 成功基準

### 5.1 完了の定義

- [ ] iPhoneでアプリが正常に起動する
- [ ] カメラで自撮り写真を撮影できる
- [ ] 撮影した写真からパーソナルカラー診断結果が表示される
- [ ] 診断結果（イエベ・ブルベ）と理由が分かりやすく表示される
- [ ] 基本的なエラーケースで適切なエラーメッセージが表示される
- [ ] App Store申請に必要な要件を満たす

### 5.2 受け入れテスト

- 小学5年生が一人で操作できる
- 友達同士で楽しく使える
- 10秒以内に診断結果が表示される
- エラーが発生しても適切にユーザーに伝わる

## 6. 想定されるリスク

### 6.1 技術的リスク

- **AI診断精度**: Gemini-2.5-proの画像解析精度が期待値を下回る可能性
  - 対策: プロンプトの最適化、複数回のテスト実行
- **ADK環境構築**: ADK + Agent Engineの環境構築に想定以上の時間がかかる可能性
  - 対策: 早期の環境構築着手、代替案の検討
- **App Store審査**: 申請時の審査で想定外の指摘を受ける可能性
  - 対策: Apple審査ガイドラインの事前確認

### 6.2 スケジュールリスク

- **短期開発期間**: 1ヶ月という限られた期間での機能実装
  - 対策: MVP機能に絞った開発、並行作業の最大化
- **テスト期間不足**: 十分なテスト期間が確保できない可能性
  - 対策: 開発と並行したテスト実行
