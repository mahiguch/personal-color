# 全年齢対応パーソナルカラー診断 - 要件定義

## 概要

現在小学5年生向けに最適化されているパーソナルカラー診断システムに、年齢と性別推定機能を追加し、全年齢層に対応したパーソナライズされた診断結果・説明文を提供するシステムを構築する。

## 背景・課題

### 現状
- パーソナルカラー診断は小学5年生向けの説明文に固定
- 年齢や性別を考慮しない一律の説明・アドバイス
- 他の年齢層には内容が適切でない可能性

### 課題
- 大人や高齢者にとって説明文が幼稚に感じられる
- 男性向けの色彩活用アドバイスが不足
- 年代別のライフスタイルに合わせた提案ができていない

## 機能要件

### 1. 年齢推定機能
- **推定方法**: Gemini Vision AIによる画像解析
- **推定精度**: 年代区分での推定（具体的年齢は推定しない）
- **年代区分**:
  - 子供（8-12歳）: 現在の小学生向けスタイル
  - 学生（13-22歳）: トレンド志向、ポップな表現
  - 社会人（23-39歳）: 実用的、プロフェッショナル
  - 中高年（40-59歳）: 上品、大人の魅力重視
  - シニア（60歳以上）: 気品、健康的な印象重視

### 2. 性別推定機能
- **推定方法**: Gemini Vision AIによる画像解析
- **性別区分**:
  - 男性: ファッション実用性重視のアドバイス
  - 女性: 詳細な色彩理論とメイク・ファッション両方のアドバイス
  - 不明: 中性的な表現で男女両方に対応

### 3. 適応的説明文生成機能
- **基本方針**: 年代・性別の組み合わせ（最大15パターン）に応じた説明文
- **対応範囲**:
  - 語調・文体の調整
  - 推奨色の活用シーン
  - ライフスタイルに合わせたアドバイス
  - メイク・ファッション提案の深度調整

### 4. 診断結果表示機能
- **基本表示**: パーソナルカラー診断結果
- **拡張表示**: 年代・性別情報（ユーザー選択制）
- **適応表示**: 年代・性別に最適化された説明文・アドバイス

## 技術要件

### 1. AI解析技術
- **使用AI**: Google Gemini Vision API
- **解析対象**: 顔画像から年代・性別・パーソナルカラーを同時分析
- **処理方式**: シングルパス処理（効率性重視）

### 2. データ構造拡張
- **新エンティティ**:
  ```
  AgeGroup: child | student | adult | middleAge | senior
  Gender: male | female | unknown
  PersonAnalysis: { ageGroup, gender, confidence }
  ```
- **既存エンティティ拡張**:
  ```
  DiagnosisResult: 既存フィールド + personAnalysis + 適応済み説明文
  ```

### 3. プロンプト設計
- **年代・性別推定プロンプト**: パーソナルカラー分析と統合
- **説明文生成プロンプト**: 年代・性別パラメータに応じた動的生成

## 非機能要件

### 1. プライバシー要件
- **推定情報制御**: 年代・性別情報の表示/非表示をユーザーが選択可能
- **データ保護**: 具体的年齢ではなく年代区分での推定
- **保存期間**: 推定情報の保存期間制限

### 2. パフォーマンス要件
- **応答時間**: 既存の診断時間（2-5秒）を維持
- **精度要件**: 年代推定 70%以上、性別推定 80%以上の精度
- **可用性**: 推定失敗時のフォールバック対応

### 3. ユーザビリティ要件
- **直感性**: 年代・性別情報の表示は分かりやすく
- **選択制**: プライバシー重視ユーザーへの配慮
- **一貫性**: 既存UI/UXとの整合性維持

## ユーザーストーリー

### 1. 子供ユーザー
```
子供として、現在と同じ分かりやすい説明で、
パーソナルカラーを楽しく学べるようにしたい。
```

### 2. 学生ユーザー
```
学生として、トレンドを意識した色の使い方や、
学校・プライベートでの活用方法を知りたい。
```

### 3. 社会人ユーザー
```
社会人として、ビジネスシーンに適した色選びと、
プライベートでのファッション提案を得たい。
```

### 4. 中高年ユーザー
```
大人として、上品で落ち着いた色使いのアドバイスと、
年代に適したファッション提案を受けたい。
```

### 5. シニアユーザー
```
シニアとして、健康的で明るい印象を与える色選びと、
品のあるスタイリング提案を知りたい。
```

### 6. 男性ユーザー
```
男性として、ビジネス・カジュアル両方で使える、
実用的な色選びのアドバイスが欲しい。
```

## 成功基準

### 1. 機能的成功基準
- [ ] 年代推定精度: 70%以上
- [ ] 性別推定精度: 80%以上  
- [ ] 全15パターンの適応的説明文生成
- [ ] 既存診断精度の維持（85%以上）

### 2. ユーザビリティ成功基準
- [ ] 年代別ユーザー満足度: 4.0/5.0以上
- [ ] 説明文の適切性評価: 4.2/5.0以上
- [ ] プライバシー設定の利用率: 30%以上

### 3. 技術的成功基準
- [ ] 診断応答時間: 5秒以内
- [ ] システム可用性: 99.5%以上
- [ ] エラー率: 1%以下

## 制約条件

### 1. 技術制約
- Gemini Vision APIの利用制限内での実装
- 既存システムアーキテクチャとの互換性維持
- Flutter/Python既存コードベースの活用

### 2. プライバシー制約
- GDPR/個人情報保護法への準拠
- 未成年者の画像処理に対する特別配慮
- ユーザーデータの最小化原則

### 3. リソース制約
- 既存開発チーム・予算内での実装
- 段階的リリース（破壊的変更の回避）

## 除外事項

### 明確に対象外とするもの
- 具体的年齢（例：23歳）の推定・表示
- 顔認識による個人特定機能
- 過去の診断履歴との比較分析
- リアルタイム年齢変化の追跡
