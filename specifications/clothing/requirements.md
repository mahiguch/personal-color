# 衣料品リコメンド機能 要件定義書

## 1. 機能概要

### 1.1 目的
パーソナルカラー診断結果に基づき、ユーザーに最適な衣料品（ファッションアイテム）を推奨する機能を提供する。

### 1.2 対象ユーザー
- パーソナルカラー診断を完了したユーザー
- 自分に似合うファッションアイテムを知りたいユーザー

### 1.3 機能概要
- 診断結果ページに「おすすめのファッション」ボタンを追加
- 衣料品リコメンドページでパーソナルカラーに基づいた商品推奨を表示
- トップス、ボトムス、アクセサリーの3カテゴリで商品を分類
- 各商品にパーソナルカラーに基づく推奨理由をAI生成で提供

## 2. ユーザーストーリー

### 2.1 基本フロー
```
診断結果ページ
    ↓ 「おすすめのファッション」ボタンタップ
衣料品リコメンドページ
    ↓ 商品タップ
Amazon商品詳細ページ（外部リンク）
```

### 2.2 ユーザーストーリー
- **As a** パーソナルカラー診断を完了したユーザー
- **I want** 自分のパーソナルカラーに似合う衣料品を知りたい
- **So that** 自分に似合うファッションで自信を持てる

### 2.3 受け入れ条件
- [ ] 診断結果ページに「おすすめのファッション」ボタンが表示される
- [ ] 既存の「おすすめのメイク」ボタンと縦に並んで配置される
- [ ] ボタンタップで衣料品リコメンドページに遷移する
- [ ] 衣料品リコメンドページでパーソナルカラーに基づいた商品が表示される
- [ ] 各カテゴリでAI生成の推奨理由が表示される
- [ ] 商品タップでAmazon商品詳細ページに遷移する

## 3. 機能要件

### 3.1 UI/UX要件

#### 3.1.1 診断結果ページの変更
- **ボタン追加**:
  - テキスト: 「おすすめのファッション」
  - アイコン: 服装関連のアイコン（例: Icons.checkroom）
  - 配置: 既存の「おすすめのメイク」ボタンの下に配置（縦並び）
  - スタイル: 既存のメイクボタンと同じデザイン仕様

#### 3.1.2 衣料品リコメンドページの新規作成
- **ページタイトル**: 「おすすめコーデ」
- **レイアウト構成**:
  - AppBar: タイトル表示、戻るボタン
  - タブ構成: トップス、ボトムス、アクセサリーの順序
  - 各タブで商品カード表示（3商品/カテゴリ）
  - AI生成の推奨理由表示エリア

#### 3.1.3 商品カード仕様
- **表示項目**:
  - 商品画像
  - 商品名
  - ブランド名  
  - 価格
  - 簡潔な説明文
- **インタラクション**:
  - タップでAmazon商品詳細ページに遷移（外部リンク）
- **デザイン**: 既存のメイクアップ商品カードと統一

### 3.2 API要件

#### 3.2.1 新規エンドポイント
```
GET /api/v1/clothing-recommendations/{personal_color_type}
```

#### 3.2.2 リクエスト仕様
- **パラメータ**: 
  - `personal_color_type`: string (spring/summer/autumn/winter)
- **認証**: 不要（既存メイクアップAPIと同様）
- **レート制限**: 60リクエスト/分（既存APIと同様）

#### 3.2.3 レスポンス仕様

### 3.3 データ要件

#### 3.3.1 商品マスタデータ
- **ファイル**: `server/data/clothing_products.json`
- **構造**: 既存の `makeup_products.json` と同様
- **カテゴリ**: 
  - `tops` (トップス: シャツ、ニット、ジャケット等)
  - `bottoms` (ボトムス: パンツ、スカート、ショーツ等)  
  - `accessories` (アクセサリー: ネックレス、イヤリング、時計、バッグ等)
- **商品数**: 各パーソナルカラー × 各カテゴリあたり3商品

#### 3.3.2 パーソナルカラー対応
- **対応シーズン**: Spring、Summer、Autumn、Winter（全4タイプ）
- **シーズン別特徴**:
  - **Spring**: 明るく鮮やかな色、軽やかな素材
  - **Summer**: 涼しげで上品な色、エレガントなデザイン
  - **Autumn**: 深みのある暖色系、リッチな素材感
  - **Winter**: クールでシャープな色、モダンなデザイン

### 3.4 AI生成要件

#### 3.4.1 推奨理由生成
- **生成対象**: 各カテゴリ（トップス、ボトムス、アクセサリー）の推奨理由
- **生成条件**: パーソナルカラータイプ × 商品情報
- **文体**: 小学5年生向けのわかりやすい表現
- **文字数**: 50-100文字程度
- **内容**: パーソナルカラーの特徴を活かした具体的な推奨理由

#### 3.4.2 AI モデル
- **使用サービス**: Vertex AI (Gemini-2.5-pro)
- **プロンプト管理**: 既存の makeup_recommendation_prompts を拡張
- **フォールバック**: AI生成失敗時の固定文言を用意

## 4. 非機能要件

### 4.1 性能要件
- **レスポンス時間**: 3秒以内（AI生成含む）
- **同時接続数**: 100ユーザー
- **可用性**: 99.9%以上

### 4.2 セキュリティ要件
- **入力検証**: パーソナルカラータイプの妥当性チェック
- **AI出力検証**: 生成内容のサニタイゼーション
- **外部リンク**: Amazon URLの検証

### 4.3 互換性要件
- **既存機能**: メイクアップリコメンド機能との共存
- **UI統一**: 既存デザインシステムとの整合性
- **データ構造**: 将来のAmazon API連携を考慮した拡張可能な設計

## 5. 制約事項

### 5.1 技術的制約
- **固定データ**: 初期段階は静的JSONファイルからの商品データ提供
- **外部連携**: Amazon商品詳細ページへの直接リンクのみ（在庫確認等なし）
- **デバイス対応**: iOS（iPhone）のみ対応

### 5.2 ビジネス制約
- **商品更新**: 手動での商品データメンテナンス
- **収益モデル**: アフィリエイトリンク等の収益化は将来検討
- **リーガル**: 商品推奨に関する免責事項の明記

## 6. 将来的な拡張性

### 6.1 Amazon API連携
- **Product Advertising API**: 商品情報の動的取得
- **在庫連携**: リアルタイム在庫状況の反映
- **価格更新**: 最新価格情報の自動取得

### 6.2 機能拡張
- **商品フィルタ**: 価格帯、ブランド、サイズでの絞り込み
- **レビュー連携**: Amazon商品レビューの表示
- **シーズン連動**: 季節に応じた商品推奨の調整

### 6.3 分析機能
- **推奨効果測定**: クリックスルー率、コンバージョン率の計測
- **ユーザー行動分析**: 商品カテゴリ別の嗜好分析
- **A/Bテスト**: 推奨ロジックの最適化
