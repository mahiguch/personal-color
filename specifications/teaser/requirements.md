# ディザーサイト 要件定義書

## 1. プロジェクト概要

### 1.1 目的
- App Store Connect申請に必要なプライバシーポリシーとサポートページのURL提供
- パーソナルカラー診断アプリの公式Webサイトとしての機能

### 1.2 背景
- iOSアプリ「パーソナルカラー診断アプリ」のApp Store申請にはプライバシーポリシーとサポートページのURLが必須
- アプリの紹介・宣伝用のWebサイトとして併用

### 1.3 スコープ
- 静的Webサイトの構築
- 必須ページ（プライバシーポリシー、サポート）の実装
- アプリ紹介ページの実装

## 2. ターゲットユーザー

### 2.1 メインターゲット
- **小学5年生**: アプリの直接利用者
- **保護者**: アプリのダウンロードと利用許可の決定者

### 2.2 利用シーン
- App Storeからアプリ詳細を確認する際
- アプリ利用中にサポートが必要な際
- プライバシーポリシーの確認が必要な際

## 3. 機能要件

### 3.1 必須ページ

#### 3.1.1 トップページ（アプリ紹介）
- **URL**: `https://[domain]/`
- **目的**: アプリの魅力を訴求し、App Storeへの誘導
- **コンテンツ**:
  - アプリ名とキャッチコピー
  - 主要機能の説明（簡単撮影、AI診断、パーソナルカラー結果）
  - アプリスクリーンショット（最低3枚）
  - 使い方の簡単な説明
  - App Storeダウンロードリンク
  - 対象年齢（小学5年生〜）の記載
  - 無料であることの強調

#### 3.1.2 プライバシーポリシー
- **URL**: `https://[domain]/privacy-policy`
- **目的**: App Store Connect申請要件の充足
- **コンテンツ**:
  - 既存の`docs/PRIVACY_POLICY.md`をベースに作成
  - 収集する情報の明記：
    - アプリの使用状況（クラッシュレポート、エラーログ）
    - 診断実行回数（統計目的）
    - デバイス情報（動作改善目的）
  - 収集しない情報の明記：
    - 撮影写真（診断後即座に削除）
    - 個人の診断結果
    - 個人情報

#### 3.1.3 サポートページ
- **URL**: `https://[domain]/support`
- **目的**: ユーザーサポートとApp Store Connect申請要件の充足
- **コンテンツ**:
  - FAQ（よくある質問）
  - お問い合わせフォーム
  - アプリの基本的な使用方法
  - トラブルシューティング

### 3.2 お問い合わせ機能
- **フォーム送信先**: mahiguch2@gmail.com
- **必須項目**:
  - 名前
  - メールアドレス
  - お問い合わせ内容
- **任意項目**:
  - デバイス情報（iOS version等）

## 4. 非機能要件

### 4.1 技術スタック
- **フレームワーク**: Next.js (React)
- **ホスティング**: Firebase App Hosting
- **スタイリング**: Tailwind CSS
- **フォーム処理**: Next.js API Routes + Cloud Functions

### 4.2 パフォーマンス要件
- **ページ読み込み時間**: 3秒以内
- **Core Web Vitals**: Good評価を維持
- **モバイルフレンドリー**: 必須

### 4.3 デザイン要件
- **デザイン方向性**: 保護者向けの信頼感のあるデザイン
- **レスポンシブ対応**: 必須（スマートフォン、タブレット、PC）
- **アクセシビリティ**: WCAG 2.1 AA準拠
- **カラーパレット**: アプリのブランドカラーと統一

### 4.4 多言語対応
- **初期対応**: 日本語のみ
- **将来対応**: 英語（必要に応じて）

### 4.5 SEO・アナリティクス
- **SEO対応**: 基本的なメタタグ設定
- **アクセス解析**: 不要（初期段階）

### 5.1 App Store情報

- **アプリアイコン**: `docs/assets/icons/*.png`を使用

### 5.2 必要URL（App Store Connect設定用）
- **マーケティングURL**: `https://[domain]/`
- **プライバシーポリシーURL**: `https://[domain]/privacy-policy`
- **サポートURL**: `https://[domain]/support`

## 6. 制約・前提条件

### 6.1 ドメイン
- 独自ドメイン取得予定
- サブドメインでの運用

### 6.2 リリース要件
- **公開予定**: 可能な限り早期（App Store申請前）
- **運用開始**: サイト公開と同時

### 6.3 保守・運用
- **更新頻度**: 低頻度（アプリアップデート時など）
- **監視**: 基本的な死活監視のみ

## 7. 成功指標

### 7.1 技術指標
- サイトの正常稼働率: 99%以上
- App Store Connect審査の通過

### 7.2 ビジネス指標
- App Storeからのサイト流入
- お問い合わせフォームの正常動作

## 8. リスクと対策

### 8.1 技術リスク
- **ホスティングサービスの障害**: Firebase App Hosting + 独自ドメインでリスク軽減
- **フォーム機能の不具合**: 事前テストとフォールバック連絡先の明記
- **Firebase設定の複雑性**: 段階的な移行とテスト環境での検証

### 8.2 運用リスク
- **お問い合わせ対応遅延**: 自動返信メールの設定
