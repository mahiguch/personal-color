## 注意事項

### 開発原則
- **1タスク = 1コミット**: 各タスク完了時に必ずコミット
- **品質優先**: 完了条件を満たすまで次タスクに進まない
- **ドキュメント更新**: 実装中の学びを設計書に反映
- **テスト駆動**: テストケースを意識した実装

### コミュニケーション
- **日次進捗**: 各タスクの完了状況を毎日確認
- **週次レビュー**: Phase 単位での品質・進捗レビュー
- **ブロッカー対応**: 依存関係での遅延を即座に報告

### 品質管理
- **コードレビュー**: 重要なタスクは Pull Request でレビュー
- **実機テスト**: iPhone実機での動作を必ず確認
- **セキュリティチェック**: 画像データ処理の安全性を重視
