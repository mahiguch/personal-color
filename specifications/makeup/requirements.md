# 要件定義書 - メイクアップ推奨機能

## 1. 目的

パーソナルカラー診断結果に基づいて、小学5年生ユーザーに適したメイクアップ商品（アイシャドウ、チーク、リップ）を推奨し、その理由をわかりやすく説明することで、安全で年齢適切な美容体験を提供する。

## 2. 機能要件

### 2.1 必須機能

#### 診断結果ページの拡張
- [ ] 既存の`IOSDiagnosisResultPage`に「おすすめのメイク」ボタンを追加
- [ ] ボタンは診断結果カードの下部、現在のアクションボタン群（再診断）の隣に配置
- [ ] Material Design 3に準拠した小学生向けの親しみやすいデザイン
- [ ] タップ時にメイクアップ推奨画面へのナビゲーション実行

#### メイクアップ推奨ページの新規作成
- [ ] 新しい`MakeupRecommendationPage`ウィジェットの実装
- [ ] 診断されたパーソナルカラータイプ（Spring/Summer/Autumn/Winter）の表示
- [ ] 3つのカテゴリ別商品推奨表示（各3商品まで）：
  - アイシャドウ（パレット商品3商品）
  - チーク（単色3商品）
  - リップ（単色3商品）
- [ ] 各商品に対する推奨理由の表示（小学5年生向け3-4文程度の中程度説明）
- [ ] パーソナルカラータイプと商品の関連性説明（中程度の詳細さ）
- [ ] 商品画像、商品名、価格をカード形式で表示
- [ ] Amazon商品ページへのリンク
- [ ] 外部サイト遷移時の確認ダイアログ表示
- [ ] 診断結果ページに戻るナビゲーション機能

#### API エンドポイントの追加
- [ ] `GET /api/v1/makeup-recommendations/{personal_color_type}` エンドポイント
- [ ] **第1段階**: 静的データベース（JSON）からの商品データ提供
- [ ] **第2段階**: Amazon Product Advertising API との統合（権限取得後）
- [ ] パーソナルカラータイプに基づいた商品フィルタリング
- [ ] 価格上限2,000円以下の商品のみ推奨
- [ ] レスポンスにGemini AIによる推奨理由を含める（3-4文の中程度説明）

### 2.2 オプション機能

- [ ] 商品のお気に入り機能（ローカル保存）
- [ ] 価格帯フィルター機能
- [ ] 商品レビュー表示
- [ ] 類似商品の代替提案機能
- [ ] Amazon Product Advertising API統合（将来実装）

## 3. 非機能要件

### 3.1 パフォーマンス

- メイクアップ推奨ページの初回読み込み時間：3秒以内
- 商品データAPI レスポンス時間：2秒以内
- 商品画像読み込み：プログレッシブローディングで1秒以内
- APIエラー時の即座エラー表示（デフォルト商品は表示しない）

### 3.2 セキュリティ

- 商品データの適切なサニタイゼーション
- 外部リンクへの遷移時の確認ダイアログ実装
- ローカルキャッシュデータの暗号化
- Gemini AI応答内容の適切性検証

### 3.3 保守性

- 既存のClean Architecture構造への適合
- 新しい`features/makeup/`モジュールとして独立実装
- 既存コードへの影響を最小限に抑制
- ユニットテスト・ウィジェットテストの実装
- 静的商品データの管理容易性

### 3.4 互換性

- 既存のiOS専用アプリとの完全互換性
- Material Design 3テーマとの統合
- 既存のDIコンテナ（get_it）との連携
- 現在のナビゲーション構造（MaterialPageRoute）との適合
- オフライン時の適切なエラー表示

## 4. 制約事項

### 4.1 技術的制約

- **Amazon API制限**: 当面は静的データで実装、将来的にAPI統合
- 商品データの更新頻度：手動更新（静的データ期間中）
- iOS専用実装（Android対応は将来実装）
- Flutter 3.13+ との互換性維持
- インターネット接続必須（オフライン時はエラー表示）

### 4.2 ビジネス制約

- 商品価格上限：2,000円以下
- 各カテゴリ最大3商品まで表示
- ブランド・成分・年齢による制限なし
- 保護者承認プロセス不要
- 税込み価格表示は必須ではない

### 4.3 UX/UI制約

- 既存のアプリテーマ（インディゴブルー・ピンク）との統一
- 小学5年生の読解力に適した文言（3-4文程度の説明）
- カード形式での商品表示
- タップしやすいボタンサイズ（最小44pt）
- アクセシビリティ対応（VoiceOver対応）

## 5. 成功基準

### 5.1 完了の定義

- [ ] 診断結果ページに「おすすめのメイク」ボタンが正常に表示され、機能する
- [ ] メイクアップ推奨ページが3つのカテゴリで各3商品を正しく表示する
- [ ] 静的商品データベースから商品データを正常に取得・表示できる
- [ ] 商品推奨理由がGemini AIにより適切に生成される（3-4文の中程度説明）
- [ ] Amazon商品ページへの遷移が確認ダイアログ付きで動作する
- [ ] ローカルキャッシュ機能が正常に動作する
- [ ] オフライン時に適切なエラー表示が行われる
- [ ] すべてのナビゲーションが正常に動作する
- [ ] ユニットテスト・ウィジェットテストが90%以上のカバレッジを達成
- [ ] パフォーマンス要件をすべて満たす
- [ ] 実機（iOS）での動作テストが完了

### 5.2 受け入れテスト

- 小学5年生ユーザーが迷うことなくメイクアップ推奨機能を利用できる
- 推奨される商品が価格制限（2,000円以下）を満たしている
- 推奨理由が年齢に適切でわかりやすい（3-4文程度）
- 外部サイトへの遷移時に適切な確認が行われる
- 既存の診断機能に影響を与えない
- アプリのパフォーマンスが劣化しない

## 6. 想定されるリスク

### 6.1 技術的リスク
- **静的データ管理**: 商品情報の更新作業負荷増加
  - **対策**: 定期的な商品データレビュープロセス確立
- **商品データ品質**: 不適切な商品情報や価格情報
  - **対策**: 商品データの品質チェック機能実装
- **パフォーマンス低下**: 商品データ読み込みによる画面遷移の遅延
  - **対策**: プログレッシブローディングとローカルキャッシング
- **Amazon API将来統合**: API権限取得・統合時の技術的課題
  - **対策**: アーキテクチャの拡張性確保

### 6.2 ビジネスリスク
- **商品情報の正確性**: 価格変動や在庫切れによる情報不整合
  - **対策**: 定期的な商品情報更新とエラーハンドリング
- **法的リスク**: Amazonとの提携関係やアフィリエイト規約
  - **対策**: 必要な法的手続きの実施
