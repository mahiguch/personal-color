# Android アプリ要件定義書

## プロジェクト概要

### 目的
既存のiOSパーソナルカラー診断アプリをAndroidプラットフォームに展開し、より多くのユーザーにサービスを提供する。

### ターゲット
- **主要ターゲット**: 小学5年生
- **アプリカテゴリ**: エンターテイメント
- **プラットフォーム**: Android（Google Play Store配信）

## 機能要件

### 1. 基本機能（iOSアプリと同等）
- **カメラ撮影機能**
  - フロントカメラでの自撮り
  - 撮影ガイダンス表示
  - 撮影した写真の確認・再撮影機能

- **パーソナルカラー診断機能**
  - 撮影画像をサーバーに送信
  - Vertex AI（Gemini-2.5-pro）による診断処理
  - 診断結果の受信・表示

- **診断結果表示機能**
  - パーソナルカラータイプの表示
  - おすすめカラーパレットの表示
  - 診断理由・解説の表示
  - 結果の保存・共有機能

### 2. プラットフォーム固有機能
- **Androidパーミッション対応**
  - カメラアクセス許可
  - ストレージアクセス許可
  - インターネットアクセス許可

- **Material Design対応**
  - Android Design Guidelinesに準拠
  - Material You（Dynamic Color）対応検討
  - Androidネイティブな操作感の提供

## 技術要件

### 1. 開発技術スタック
- **フレームワーク**: Flutter 3.13+
- **言語**: Dart 3.0+
- **アーキテクチャ**: Clean Architecture + DDD（iOSアプリと共通）
- **コード共有**: 既存iOSアプリとの最大限の共有

### 2. サーバー連携
- **API**: 既存Python サーバー（FastAPI）を利用
- **AI処理**: Vertex AI（Gemini-2.5-pro）
- **通信**: HTTP REST API
- **画像送信**: Base64エンコード形式

### 3. Android固有技術要件
- **最小SDK**: Android 7.0（API Level 24）
- **ターゲットSDK**: 最新安定版
- **アーキテクチャ**: arm64-v8a, armeabi-v7a
- **パーミッション**: 必要最小限の権限要求

## 非機能要件

### 1. パフォーマンス
- **起動時間**: 3秒以内
- **診断処理時間**: 10秒以内（ネットワーク状況による）
- **メモリ使用量**: 100MB以下
- **バッテリー消費**: 最小化

### 2. ユーザビリティ
- **対象年齢適応**: 小学5年生でも直感的に操作可能
- **アクセシビリティ**: Android Accessibility Service対応
- **言語**: 日本語のみ
- **オフライン対応**: 診断機能以外は可能な限り対応

### 3. セキュリティ
- **個人情報保護**: 撮影画像の適切な取り扱い
- **通信暗号化**: HTTPS通信
- **データ保存**: ローカルストレージの暗号化検討

## UI/UX要件

### 1. デザイン方針
- **Material Design 3**: 最新のMaterial Design適用
- **色彩**: パーソナルカラー診断アプリにふさわしい配色
- **タイポグラフィ**: 小学生にも読みやすいフォント選択
- **アニメーション**: 楽しさを演出する適度なアニメーション

### 2. ユーザーフロー
1. アプリ起動 → オンボーディング（初回のみ）
2. カメラ画面 → 撮影ガイド → 撮影実行
3. 撮影確認 → 診断開始 → ローディング
4. 診断結果表示 → 詳細確認 → 保存・共有

### 3. レスポンシブ対応
- **画面サイズ**: 5インチ〜7インチのスマートフォン対応
- **画面向き**: 主にポートレート、一部ランドスケープ対応
- **解像度**: hdpi〜xxxhdpi対応

## リリース要件

### 1. Google Play Store対応
- **アプリ情報**
  - アプリ名: 「パーソナルカラー診断」（仮）
  - カテゴリ: エンターテイメント
  - 年齢制限: 3歳以上
  - 価格: 無料

- **ポリシー準拠**
  - Google Play Developere Policy準拠
  - 子供向けアプリのポリシー適用
  - プライバシーポリシーの設置
  - データ収集・利用の透明性確保

### 2. 審査対策
- **コンテンツ**: 教育的・エンターテイメント性を重視
- **広告**: なし（初期リリース）
- **課金**: なし（初期リリース）
- **権限**: 必要最小限（カメラ、インターネット）

## 制約事項

### 1. 技術的制約
- **既存サーバー**: 現在のPythonサーバーをそのまま利用
- **AI精度**: Vertex AIの診断精度に依存
- **ネットワーク**: オンライン診断が必須

### 2. ビジネス制約
- **開発期間**: 段階的リリースによる早期市場投入
- **保守性**: iOSアプリとの共通化によるメンテナンス効率化
- **拡張性**: 将来的な機能追加に対応可能な設計
