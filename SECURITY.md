## 脆弱性の報告

セキュリティ脆弱性を発見した場合は、以下の手順に従って報告してください。

### 報告方法

1. **公開Issueでは報告しないでください** - セキュリティ問題は非公開で報告してください
2. メールで連絡してください: `security@your-domain.com`
3. 以下の情報を含めてください：
   - 脆弱性の詳細な説明
   - 再現手順
   - 影響の範囲
   - 可能であれば修正案

### 対応プロセス

1. **確認** - 24時間以内に受信確認を送信
2. **調査** - 7日以内に初期評価を完了
3. **修正** - 重要度に応じて修正スケジュールを決定
4. **公開** - 修正完了後、適切なタイミングで脆弱性情報を公開

## セキュリティベストプラクティス

### 開発者向け

#### 機密情報の管理
- **.env ファイルを絶対にコミットしない**
- API キー・認証情報は環境変数で管理
- Firebase設定ファイルは `.example` ファイルのみコミット

#### コード品質
- 定期的なセキュリティ監査
- 依存関係の脆弱性チェック
- 最小権限の原則の適用

### 利用者向け

#### アプリ利用時
- 最新バージョンの使用
- 公式App Store/Google Play からのみダウンロード
- 不審な動作の報告

#### プライバシー保護
- 撮影画像は診断後即座に削除
- 個人情報は一切収集・保存されない
- 子ども向けアプリとして安全設計

## インシデント対応

重大なセキュリティインシデントが発生した場合：

1. **即座に対処** - サービス停止も含む緊急対応
2. **ユーザー通知** - 影響を受けるユーザーへの迅速な通知
3. **詳細調査** - 根本原因の特定と対策実施
4. **事後報告** - 透明性のある事後報告書の公開

## セキュリティ連絡先

- **緊急時**: `security@your-domain.com`
- **一般問い合わせ**: `support@your-domain.com`
