# App Store スクリーンショット撮影ガイド

## 📱 必要なスクリーンショット仕様

### デバイスサイズ要件

#### iPhone必須サイズ
- **iPhone 6.7"** (iPhone 15 Pro Max, 14 Pro Max): **1290×2796 px**
- **iPhone 6.5"** (iPhone 14 Plus, 13 Pro Max): **1284×2778 px**

#### iPhone推奨追加サイズ  
- **iPhone 6.1"** (iPhone 15, 14, 13): **1179×2556 px**
- **iPhone 5.5"** (iPhone 8 Plus): **1242×2208 px**

### ファイル仕様
- **ファイル形式**: PNG または JPG
- **色空間**: sRGB または P3
- **透明度**: 不可
- **最大ファイルサイズ**: 各10MB

## 📸 撮影する画面（5画面）

### 1. ホーム画面（ウェルカム画面）
**目的**: アプリの概要とスタートボタン

**撮影内容:**
- アプリタイトル「パーソナルカラー診断アプリ」
- 簡単な説明テキスト「AIであなたに似合う色を診断」
- 大きな「診断開始」ボタン
- 親しみやすいカラフルなデザイン

**撮影ポイント:**
- 明るく親しみやすい印象
- 小学生でも理解しやすい文字とアイコン
- カラフルでわくわくする配色

### 2. カメラ撮影画面
**目的**: 撮影手順の分かりやすさ

**撮影内容:**
- カメラプレビュー画面
- 撮影ガイド（顔の位置を示すフレーム）
- 「撮影」ボタン
- 分かりやすい操作説明

**撮影ポイント:**
- プライバシーに配慮（実際の顔は使用しない）
- イラストやモザイク処理で代替
- 撮影手順が明確に分かる表示

**⚠️ プライバシー注意事項:**
```
- 実際の人物の顔は使用禁止
- イラストやアバター画像で代替
- または適切なモザイク・ぼかし処理
```

### 3. 診断中画面（ローディング）
**目的**: AIが分析中である ことの表示

**撮影内容:**
- ローディングアニメーション
- 「AI が診断中...」のメッセージ
- 進行状況インジケーター
- 診断プロセスの説明

**撮影ポイント:**
- AI技術の先進性を表現
- 安心感を与えるデザイン
- 待ち時間を楽しくする工夫

### 4. 診断結果画面
**目的**: イエベ・ブルベ結果の表示

**撮影内容:**
- 大きく「イエローベース」または「ブルーベース」の表示
- 診断理由の説明文
- 結果に合わせた背景色
- 次のステップへのボタン

**撮影ポイント:**
- 結果が一目で分かる大きな表示
- 理由が分かりやすく説明されている
- ポジティブで明るい印象

### 5. カラーパレット画面
**目的**: おすすめ色の表示

**撮影内容:**
- 診断結果に基づく推奨カラーパレット
- 各色の名前や説明
- 実用的なアドバイス
- 「もう一度診断」ボタン

**撮影ポイント:**
- 美しいカラーパレット表示
- 実用的で参考になる情報
- 継続利用を促す要素

## 🎬 App Preview（プレビュー動画）仕様

### 動画仕様
- **時間**: 15-30秒（推奨: 20秒）
- **解像度**: スクリーンショットと同じ
- **フレームレート**: 30fps
- **ファイル形式**: MOV または MP4
- **最大ファイルサイズ**: 500MB
- **音声**: なし（無音推奨）

### 動画構成案（20秒）

```
0-2秒:   アプリアイコン + タイトル表示
3-5秒:   ホーム画面 → 「診断開始」タップ
6-10秒:  カメラ画面 → 撮影 → 診断中
11-15秒: 診断結果表示 → カラーパレット表示  
16-20秒: まとめ画面 + アプリロゴ
```

### 動画撮影のポイント

**画面遷移の流れ:**
1. **スムーズな画面切り替え**
2. **タップ操作の視覚的表示**
3. **診断プロセスの自然な流れ**
4. **結果表示の魅力的な演出**

**技術的ポイント:**
- 実機での録画（シミュレーターより滑らか）
- 安定した手ぶれのない撮影
- 適切な画面の明るさ設定

## 🛠 撮影ツール・方法

### iOS シミュレーター使用
```bash
# Xcodeシミュレーターで撮影
# Device > Screenshot で保存
# 自動的に適切な解像度で保存される
```

### 実機撮影
```bash  
# Flutter コマンドで実機に接続
flutter devices
flutter run -d [device-id]

# 実機の電源ボタン + ボリュームアップでスクリーンショット
```

### 画面録画（App Preview用）
```bash
# macOS QuickTime Player を使用
# 新規画面収録 > iOS デバイス選択
# または Xcode Simulator の画面収録機能
```
